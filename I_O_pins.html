ESP32 Pin Configuration and Power Supply Guide

Complete reference for ESP32 GPIO pins, input/output functions, and power options

ESP32 Development Board Pinout

The ESP32 microcontroller has a rich set of GPIO (General Purpose Input/Output) pins that can be configured for various functions. Understanding the pinout is essential for connecting components correctly.

Key Features of ESP32 Pins:

  • Up to 36 GPIO pins (depending on the module)
  • Digital input and output capabilities
  • Analog-to-Digital Converter (ADC) on 18 pins
  • Digital-to-Analog Converter (DAC) on 2 pins
  • Pulse Width Modulation (PWM) on all digital pins
  • Touch sensor capability on 10 pins
  • Multiple communication protocols (I2C, SPI, UART, I2S)

Digital Input/Output Pins

Digital pins can be configured as either inputs or outputs. As inputs, they read HIGH (3.3V) or LOW (0V). As outputs, they can source or sink current.

Digital I/O Example 
from machine import Pin

# Configure GPIO2 as output
led = Pin(2, Pin.OUT)

# Configure GPIO4 as input with pull-up resistor
button = Pin(4, Pin.IN, Pin.PULL_UP)

# Set output HIGH (3.3V)
led.value(1)

# Read input state
button_state = button.value()

Important Notes for Digital I/O:

  • ESP32 GPIO pins operate at 3.3V logic levels (not 5V tolerant)
  • Maximum current per pin: 40mA (12mA recommended for safety)
  • Total current for all pins: 200mA
  • Use pull-up or pull-down resistors for stable input readings

Analog Input/Output Pins

ESP32 has built-in ADC (Analog to Digital Converter) and DAC (Digital to Analog Converter) capabilities.

Analog Input (ADC)

18 pins support 12-bit ADC with 0-3.3V range (4096 values from 0 to 4095).

ADC Example 
from machine import Pin, ADC

# Create ADC object on GPIO36
adc = ADC(Pin(36))

# Configure ADC attenuation for 0-3.3V range
adc.atten(ADC.ATTN_11DB)

# Read analog value (0-4095)
value = adc.read()
voltage = value * 3.3 / 4095  # Convert to voltage

Analog Output (DAC)

2 pins support 8-bit DAC with 0-3.3V range (256 values from 0 to 255).

DAC Example 
from machine import Pin, DAC

# Create DAC object on GPIO25
dac = DAC(Pin(25))

# Write analog value (0-255)
dac.write(128)  # Output approximately 1.65V

Communication Protocol Pins

ESP32 supports multiple communication protocols through specific pin assignments.

I2C (Inter-Integrated Circuit)

Two-wire communication protocol. Default pins: GPIO21 (SDA), GPIO22 (SCL)

I2C Example 
from machine import Pin, SoftI2C

# Create I2C object
i2c = SoftI2C(scl=Pin(22), sda=Pin(21), freq=100000)

# Scan for I2C devices
devices = i2c.scan()

SPI (Serial Peripheral Interface)

High-speed serial communication. Default pins: Various combinations available.

SPI Example 
from machine import Pin, SPI

# Create SPI object
spi = SPI(1, baudrate=1000000, polarity=0, phase=0,
          sck=Pin(14), mosi=Pin(13), miso=Pin(12))

UART (Universal Asynchronous Receiver/Transmitter)

Serial communication. Default pins: GPIO1 (TX), GPIO3 (RX)

UART Example 
from machine import UART

# Create UART object
uart = UART(1, baudrate=9600, tx=Pin(1), rx=Pin(3))

# Send data
uart.write("Hello ESP32")

# Read data
data = uart.read()

Special Function Pins

Some pins have special purposes or restrictions.

Touch Sensor Pins

10 pins support capacitive touch sensing.

Touch Sensor Example 
from machine import TouchPad, Pin
import time

# Create touch object on GPIO4
touch = TouchPad(Pin(4))

while True:
    # Read touch value (lower = touched)
    value = touch.read()
    if value < 100:  # Threshold depends on setup
        print("Touched!")
    time.sleep(0.1)

Input-Only Pins

GPIO 34, 35, 36, 39 can only be used as inputs (no output capability).

Boot Mode Pins

GPIO 0, 2, 12, 15 affect boot mode. Avoid using these for critical functions.

ESP32 Pin Usage Best Practices

‚úÖ Do:

  • Use GPIO pins 4, 5, 12-19, 21-23, 25-27, 32-33 for general purposes
  • Add pull-up resistors for buttons and switches (or use internal pull-ups)
  • Use current-limiting resistors for LEDs (220-470Œ©)
  • Keep digital signals away from analog inputs to reduce noise
  • Use the 3.3V power pin for sensors and peripherals

‚ùå Don't:

  • Apply voltages higher than 3.3V to any GPIO pin
  • Exceed 12mA per pin or 200mA total for all pins
  • Use strapping pins (0, 2, 4, 5, 12, 15) for critical functions
  • Connect heavy loads directly to GPIO pins (use transistors or drivers)
  • Leave floating inputs - always use pull-up or pull-down resistors
